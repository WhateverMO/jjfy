## 核心评估工作流程

主要评估过程由 [eval.py](/back/eval/eval.py#L31-L172) 中的 `eval_qa()` 函数处理。以下是使用方法：

### 输入数据结构

系统要求问卷和分层结构具有特定的输入格式：

| 参数 | 描述 | 示例格式 |
| --- | --- | --- |
| `questionnaire_1_counts` | 主问卷响应 | `[8, 2, 28, 69, ...]` |
| `questionnaire_2_counts` | 副问卷响应 | `[1, 2, 2, 4, ...]` |
| `q1_section_1` | 1-2 级指标模块划分 | `[8, 3, 4, 2, 2, 4, 4, 3, 3]` |
| `q1_section_2` | 3 级指标模块划分 | `[3, 2, 3, 3, 3, 2, 3, 3, 1, 2, ...]` |
| `l1_names` | 1 级指标名称 | `["A", "B", "C"]` |
| `l2_names` | 2 级指标名称 | `[["A1", "A2", "A3"], ["B1", "B2"], ...]` |
| `l3_names` | 3 级指标名称 | `[["A11", "A12"], ["A21", "A22"], ...]` |
| `q2_section` | 问卷 2 模块划分 | `[[1,2,3], [4,6], [5,7], ...]` |
| `method` | 权重计算方法 | `"bwm"` 或 `"entropy"` |

### 运行评估


```python
from eval.eval import eval_qa
from eval.test_case import qa1_case1, qa2_case1, SECTION_INDICES
from eval.generate_dict_main import level1_names, level2_names, level3_names
 
# 使用 BWM 方法执行评估
score_dict = eval_qa(
    questionnaire_1_counts=qa1_case1,
    questionnaire_2_counts=qa2_case1,
    q1_section_1=[8, 3, 4, 2, 2, 4, 4, 3, 3],
    q1_section_2=[3, 2, 3, 3, 3, 2, 3, 3, 1, 2, 2, 2, 2, 2, 2, 2, 2, 2, 3, 2, 3, 3, 2, 2, 3],
    l1_names=level1_names,
    l2_names=level2_names,
    l3_names=level3_names,
    q2_section=SECTION_INDICES,
    method="bwm"
)
 
print("评估完成。结果:", score_dict)
```

## Django 后端集成

系统包含 Django Web 框架，用于 API 访问和数据库管理：

### 启动开发服务器

```bash
# 导航到后端目录
cd back
 
# 运行 Django 开发服务器
python manage.py runserver 0.0.0.0:8000
```

服务器将在 `http://localhost:8000` 上可访问，具有以下关键功能：

-   **API 端点**：用于问卷处理的 RESTful 接口
-   **数据库管理**：带有 Django ORM 的 SQLite 数据库
-   **ER 图生成**：可视化数据库架构表示
-   **管理界面**：内置 Django 管理后台用于数据管理

### 数据库配置

系统默认使用 SQLite，如 [settings.py](/back/mysite/settings.py#L76-L80) 中配置，无需外部数据库设置即可轻松开始使用。

## 评估方法

系统支持多种权重计算算法：

| 方法 | 描述 | 使用场景 |
| --- | --- | --- |
| **BWM** (最优最劣法) | 基于成对比较的权重计算 | 当有专家判断可用时 |
| **熵权法** | 基于信息论的客观赋权 | 当倾向于数据驱动的权重时 |

> [!NOTE]
> 当你掌握指标重要性的专业知识时选择 BWM 方法，或使用熵权法进行纯数据驱动的评估。
>
